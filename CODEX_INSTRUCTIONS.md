# Инструкции для Codex по проекту ВКР

Перед любыми изменениями обязательно прочитай VKR_CONTEXT.md.

Проект: веб-система формирования фонда оценочных средств по дисциплинам базовой кафедры.

Стек:
- PHP
- PostgreSQL
- HTML/CSS
- Bootstrap/MDB
- MVC

Главное правило:
не переписывать проект с нуля, а вносить точечные изменения в существующие файлы.

Если меняешь код:
1. сначала объясни, какие файлы будут изменены;
2. меняй только нужные участки;
3. не удаляй уже работающую логику;
4. сохраняй MVC-структуру;
5. учитывай текущую БД PostgreSQL;
6. после изменения кратко напиши, что проверить.

Важная логика:
- преподаватель работает с заданиями и наборами;
- администратор только управляет правами;
- обучающийся проходит контроль;
- задание может входить в несколько наборов через task_set_items;
- нельзя ломать уже реализованные модули заданий, наборов, попыток, ответов и проверки.
