# Контекст проекта ВКР

## Тема ВКР

Разработка системы формирования фонда оценочных средств по дисциплинам базовой кафедры.

---

# Общая информация о проекте

Разрабатывается веб-система для:
- хранения заданий;
- создания и редактирования заданий;
- организации заданий по дисциплинам;
- объединения заданий в наборы;
- проведения контроля знаний;
- хранения ответов обучающихся;
- просмотра результатов;
- предоставления обратной связи;
- использования LLM как вспомогательного инструмента.

---

# Пользователи системы

## Преподаватель
Основной пользователь системы.

Может:
- создавать задания;
- редактировать задания;
- удалять задания;
- формировать наборы заданий;
- запускать контроль знаний;
- проверять ответы;
- оставлять комментарии;
- просматривать результаты;
- использовать LLM как вспомогательный инструмент.

## Администратор
НЕ работает напрямую с заданиями.

Может:
- управлять правами преподавателей;
- выдавать доступ к:
  - заданиям;
  - папкам;
  - дисциплинам;
  - наборам заданий.

## Обучающийся
Может:
- проходить контроль знаний;
- просматривать результаты;
- просматривать свои ответы;
- получать комментарии преподавателя.

---

# Важная бизнес-логика

- преподаватель по умолчанию редактирует только свои задания;
- преподаватель может использовать чужие задания в наборах, но не может их изменять без прав;
- администратор управляет доступом преподавателей;
- одно задание может входить в несколько наборов;
- связь заданий и наборов реализуется через таблицу task_set_items;
- задания с программным кодом считаются частным случаем открытого ответа;
- LLM не принимает окончательное решение, а выполняет вспомогательный анализ;
- импорт должен поддерживать вариант без ответов;
- фиксируется время ответа на каждый вопрос.

# Как работать с пользователем

Пользователь:
- плохо ориентируется в коде;
- предпочитает пошаговые инструкции;
- любит точные указания:
  - какой файл открыть;
  - после какой строки вставлять код;
  - что заменить полностью.

Если пользователь просит:
- полный файл → лучше присылать полный файл;
- диаграмму → по умолчанию нужен draw.io;
- объяснение ошибки → объяснять простым языком.
